Frequently Asked Questions

1. the difference between var, let, and const
In JavaScript, var, let, and const are used to declare variables, but they behave differently in terms of scope, reassignability, and hoisting.
2. the difference between map(), forEach(), and filter()
The map() method transforms each element and returns a new array, while filter() returns a new array with elements that meet a condition. The forEach() method executes a function on each element but does not return a new array.
3. explain arrow functions and how they are different from regular functions
Arrow functions in JavaScript provide a shorter syntax for writing functions and behave differently from regular functions in terms of how they handle the this keyword.
4. how JavaScript Promises work
In JavaScript, Promises are used to handle asynchronous operations. They represent a value that may not be available yet, but will be resolved in the future (either successfully or with an error).
5. how closures work in JavaScript
In JavaScript, closures are a fundamental concept where a function retains access to its lexical scope, even when the function is executed outside of that scope. This happens because functions in JavaScript "remember" the environment in which they were created, including variables that were in scope at the time.
